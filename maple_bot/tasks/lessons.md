# 교훈 기록 (Lessons Learned)

> 수정을 받을 때마다 패턴을 기록. 같은 실수를 반복하지 않기 위해.

---

## 2025-05 — YOLO 거짓말탐지기 파이프라인

### L1. 재감지 실패 = 화면 전환, return False 금지

**상황.** 백그라운드 스레드가 T=0에 거짓말탐지기를 감지하고 `_safety_pending = "lie"` 를 세팅. 메인 루프가 T≈1초 후 `_check_lie_detector`를 호출할 때는 이미 화면이 "투명 도형 찾기" 팝업으로 전환되어 있어서 YOLO 재감지가 실패 → 기존 코드는 `return False`로 조기 종료 → 알림·해제가 아예 실행되지 않음.

**교훈.** 배경 스레드 감지와 메인 루프 재감지 사이에는 화면 전환 시간이 존재한다. 재감지 실패를 "미감지"로 처리하면 안 된다. 재감지 실패 시 `detect_mode` 메시지만 남기고 이후 알림·해제 흐름을 그대로 진행할 것.

---

### L2. board_roi는 항상 비율 좌표로 저장

**상황.** `board_roi`가 픽셀 좌표(`client_x=32737, client_y=32348`)로 깨진 채 저장되어 있었음. 창 위치나 해상도가 바뀌면 무용지물이 됨.

**교훈.** 게임 창 내부의 영역 좌표는 반드시 비율(`x_ratio, y_ratio, w_ratio, h_ratio`)로 저장. `resolve_region_coords()`가 런타임에 `GetClientRect()` 기반으로 실제 픽셀로 변환해준다. 픽셀 좌표 직접 저장은 금지.

---

### L3. UI 경로가 사라진 코드는 dead code

**상황.** `_solve_lie_detector_manual()`은 `dialog_lie_setup.py`의 ②~⑤ 버튼이 UI에서 제거된 순간부터 `has_manual`이 항상 `False`가 되어 실행 불가능한 코드가 됨. 그럼에도 수백 줄이 남아 있어 가독성을 해침.

**교훈.** UI에서 설정 경로가 제거된 기능 코드는 즉시 같이 삭제. "나중에 쓸 수도 있으니" 남겨두지 말 것. `has_manual` 같은 플래그가 항상 `False`라면 해당 분기 전체가 dead code.

---

### L4. 같은 목적의 입력 필드는 하나로 통일

**상황.** YOLO 모델 경로 입력이 거짓말탐지기용·투명도형용·YOLO11 몬스터용으로 3곳에 분산. 거짓말탐지기와 투명도형은 실제로 같은 모델 파일을 공유하므로 2개는 항상 동기화 문제를 유발.

**교훈.** 동일 파일을 가리키는 설정값은 UI 입력도 하나로 통합하고, 저장 시 관련 config 키에 동시에 기록. 입력이 중복되면 사용자가 한쪽만 바꾸는 실수가 생긴다.

---

### L5. 알림 파이프라인은 end-to-end로 추적

**상황.** 텔레그램 알림과 경보음이 YOLO 감지 후 울리지 않음. 원인은 L1의 `return False`였지만, 처음엔 텔레그램 설정이나 토큰 문제로 오해할 뻔함.

**교훈.** 알림이 안 온다고 보고받으면, 설정값보다 **코드 흐름**을 먼저 추적. `_check_lie_detector` → 알림 코드 도달 여부 → 실제 전송 호출 순으로 역추적.

---

## 2026-05 — 잡템 판매 / UI 하드코딩 / 좌표계

### L6. 템플릿 오매칭 원인은 항상 "잘못된 상태에서 캡처"

**상황.** `scroll_bottom.png`가 기타탭이 열리자마자 오매칭되어 아이템을 하나도 판매하지 않고 종료. 원인: 스크롤 최하단이 아닌 상태에서 템플릿을 캡처했거나, 다른 UI 요소와 시각적으로 유사한 영역을 캡처.

**교훈.** 템플릿 오매칭 발생 시 threshold를 낮추거나 코드를 수정하기 전에 **템플릿 이미지 자체를 확인**할 것. 올바른 게임 상태(실제 최하단에 스크롤된 상태)에서 재캡처하는 것만으로 해결되는 경우가 많다.

---

### L7. 로그 필터는 버퍼 기반 재렌더링으로 구현

**상황.** 로그 필터 체크박스가 이후 들어오는 메시지에만 적용되고, 이미 표시된 메시지에는 소급 적용되지 않음.

**교훈.** 로그 메시지는 `_log_buffer`에 `(category, timestamp, msg)` 형태로 항상 저장하고, 체크박스 `stateChanged` 시그널에서 `_rerender_log()`를 호출해 버퍼 전체를 재렌더링. `QTextEdit`에 직접 append하는 것만으로는 소급 필터링 불가.

---

### L8. 동기 blocking 호출은 메인 루프 앞에서 반드시 async로 분리

**상황.** `_fire_lie_alarm()`이 `winsound.Beep × 3 ≈ 1.2초` blocking으로 실행되어, 알람이 끝날 때까지 투명 도형 마우스 추적이 시작되지 않음.

**교훈.** 경보음·알림 전송 등 시간이 걸리는 side effect는 반드시 `threading.Thread(daemon=True)`로 분리. 메인 루프 또는 이벤트 핸들러 안에서 blocking sleep/IO를 직접 호출하면 다음 처리가 그만큼 지연된다.

---

### L9. YOLO 모델은 학습 대상별로 분리 필요

**상황.** 사용자가 YOLO11 몬스터 감지 설정에 거짓말탐지기 학습 모델 경로(`transparent_game_v1-28`)를 그대로 입력. 다른 클래스를 학습한 모델은 대상 클래스를 감지하지 못한다.

**교훈.** YOLO 모델은 학습된 클래스(거짓말탐지기, 투명도형, 몬스터)가 다르면 절대 공유 불가. UI에서 각 용도 모델 경로를 명확히 분리해야 하며, placeholder 텍스트에 용도를 명시할 것. 몬스터 감지용 모델은 몬스터 이미지로 별도 학습이 필요.

---

### L10. UI 잠금은 locked 파라미터 + setEnabled(False)로 일관성 있게

**상황.** 설정이 완료된 섹션의 캡처/지정/✕ 버튼을 실수로 누르면 기껏 맞춘 좌표가 초기화될 위험 존재.

**교훈.** 한 번 설정하면 변경이 불필요한 UI 행은 `_add_junk_coord_row(locked=True)` 패턴으로 버튼만 제거하고 라벨은 유지. 체크박스는 `setEnabled(False)`. 완전히 숨기지 않고 현재값을 읽기 전용으로 보여주는 것이 사용자 혼란을 줄인다.
